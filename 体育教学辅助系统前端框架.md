# 体育教学辅助系统前端框架

## 1. 项目概述

体育教学辅助系统是专为体育老师设计的综合教学辅助管理平台，以学生数据管理系统为核心基础，集成体育课考勤、学生体测数据、运动会编排、体育社团活动和班级赛管理等功能模块，为体育教师提供全方位的数字化教学管理解决方案。

## 6. 状态管理

### 6.1 Redux Store结构

```
store/
├── authSlice.js       # 认证状态管理
├── dataSlice.js       # 数据状态管理
└── index.js           # Store配置
```

### 6.2 authSlice
- 管理用户认证状态
- 包含登录、注册、登出功能
- 存储用户信息和令牌

### 6.3 dataSlice
- 管理系统核心数据
- 包含学生数据、班级数据、学校信息
- 支持数据导入导出
- 提供数据操作的异步thunk

## 7. 路由配置

### 7.1 路由结构

| 路由类型 | 路径 | 组件 | 权限 |
|----------|------|------|------|
| 公共路由 | /login | Login | 无 |
| 公共路由 | /register | Register | 无 |
| 受保护路由 | / | Dashboard | 需登录 |
| 受保护路由 | /school-info | SchoolInfo | 需登录 |
| 受保护路由 | /classes | Classes | 需登录 |
| 受保护路由 | /students | Students | 需登录 |
| 受保护路由 | /users | Users | 需登录 |

### 7.2 路由实现

使用React Router的`Routes`和`Route`组件配置路由，通过`ProtectedRoute`组件实现路由保护。

## 8. 数据流设计

### 8.1 数据流程

1. 用户操作触发组件状态更新
2. 组件调用Redux action
3. Redux thunk处理异步操作（如API请求）
4. API请求返回数据
5. Reducer更新Store状态
6. 组件通过useSelector获取最新状态
7. 组件重新渲染

### 8.2 API交互

- 使用Axios进行HTTP请求
- 统一处理API响应和错误
- 支持请求拦截和响应拦截

- 使用ESLint进行代码质量检查

- 组件命名采用PascalCase
- 函数命名采用camelCase
- 常量命名采用UPPER_CASE
